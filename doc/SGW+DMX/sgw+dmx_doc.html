Introducción DocV 2.3 (InformATE!)

 

La extensión Simple Glulx Wrapper con Damusix (SGW+DMX) es una modificación de la extensión SGW v1.6.1, desarrollada por Alessandro Schillaci (con aportaciones de Vincenzo Scarpa y Paolo Maroncelli). También incluye buenas ideas y un poco de código de la versión para InformATE! de SGW, desarrollada por Mapache y Depresiv.

 

Esta versión modificada existe debido a los deseos de algunos amigos del CAAD (el grupo hispano de Ficción Interactiva) de tener la posibilidad de usar SGW (por su notable facilidad de uso) junto con mi extensión Damusix, un potente Administrador Unificado de Sonido en Glulx.

 

Tanto la versión original de SGW como Damusix hacen uso de un par de Puntos de Entrada Glk que son necesarios para el correcto funcionamiento de ambas extensiones. La versión original de SGW implementa estos puntos de entrada de una manera “tiránica” (jeje! =P) desde su propio código, sin posibilidad de alteraciones a menos que se “hackee” la extensión. Damusix, por otro lado, ocupa un “método amigable” de implementación de código para estos Puntos de Entrada Glk, usando para ello una extensión de soporte llamada Dainunek. La implementación “tiránica” de los puntos de entrada en la versión original de SGW, provoca que tanto ésta como Damusix sean extensiones totalmente incompatibles.

 

La extensión Dainunek fue originalmente escrita para Damusix con el fin de crear un sistema de Inicialización Unificada de Extensiones en Glk para permitir que varias extensiones que ocupan los Puntos de Entrada Glk puedan usarse en conjunto, sin que el programador tenga que re-escribir dichos puntos de entrada combinando el código que cada extensión utiliza para sus propios fines. Dainunek se encarga de “colectar” todo el código de los Puntos de Entrada Glk que cada extensión implementa y los ejecuta de manera unificada. Algo en verdad muy práctico.

 

SGW+DMX ocupa la extensión Dainunek para implementar su código que debe ejecutarse en los Puntos de Entrada Glk. Gracias a este simple cambio, ya se puede utilizar en conjunto con Damusix. Esto supone una gran ventaja, puesto que las funciones nativas de audio de SGW+DMX (y de la versión original de SGW) son muy básicas. Usando Simple Glulx Wrapper con Damusix puedes utilizar el potente Gestor de Audio de Damusix para hacer cosas realmente muy interesantes y mucho más avanzadas.

 

Adicionalmente, SGW+DMX trae una serie de mejoras y correcciones de Bugs respecto de la SGW original.

Comenzando

 

Simple Glulx Wrapper con Damusix requiere para funcionar correctamente:

 

  • Compilador Inform v6.30 o superior
  • Librería Española InformATE! 6/10
  • Librería “infglk.h”

 

Como ves, no es necesario tener Damusix para usar Simple Glulx Wrapper con Damusix, jejeje! =D No, Damusix no es un requisito. Si no la incluyes, SGW+DMX compilará correctamente y podrás ocupar sus funciones nativas de audio. Si incluyes adicionalmente Damusix, entonces SGW+DMX inhabilitará todas sus funciones nativas de audio y usará en su lugar las que Damusix proporciona. Evidentemente, en el código de tu juego ya no podrás utilizar las funciones de SGW+DMX para manejar el audio, sino que deberás ocupar las rutinas propias de Damusix (no te asustes, son fáciles de ocupar... además, ¿no es acaso ése el propósito de esta versión modificada? =D).

 

Entonces, si vas a usar también Damusix, estos son los requisitos adicionales:

 

  • Extensión Damusix (para InformATE!)
  • Extension Dainunek (viene incluida con Damusix)

 

 

Para qué sirve SGW+DMX

 

SGW+DMX es un pequeño y sencillo wrapper para Glulx, especialmente creado para programadores con un nivel novato en Inform-Glulx.

 

Con esta extensión es posible mostrar una imagen o simplemente tocar un sonido o música, usando un reducido conjunto de rutinas que “envuelven” las verdaderas funciones de la API Glk de Glulx. Adicionalmente se puede especificar y utilizar los diferentes estilos de texto que Glulx provee.

 

SGW+DMX es tan simple de usar que fácilmente se puede “actualizar” una aventura escrita originalmente para la Máquina-Z a una aventura con soporte multimedia para la moderna Máquina Glulx.

 

Lo mejor de todo es que utilizando SGW+DMX se puede programar una aventura que compile tanto para Glulx como para la Máquina-Z, sin apenas variar el código. Si se utilizan sólo las funciones provistas por esta extensión, no será necesario modificar nada para obtener código biplataforma.

 

SGW+DMX ha sido concebida pensando especialmente para su uso junto con la extensión Damusix. Así podrás dotar a tu juego de funcionalidades de audio muy potentes a la vez que bien sencillas de utilizar.

 

La extensión SGW+DMX implementa las siguientes características:

 

  • 2 Ventanas Gráficas:
    - una para mostrar imágenes grandes (Ej.: de la localidad actual)
    - una para mostrar imágenes pequeñas (Ej.: descripción de objetos)

 

  • 3 Canales de Audio:
    - music: canal para la música de fondo
    - chan1: canal 1 para los efectos de sonido
    - chan2: canal 2 para los efectos de sonido

 

Si usas Damusix junto con SGW+DMX, los tres canales de audio antes mencionados ya no serán creados y todas las funciones relacionadas con el sonido de SGW+DMX serán inhabilitadas. Entonces ya podrás manejar el sistema de audio completo directamente con Damusix.

 

 

Cómo usar SGW+DMX

 

Tan sencillo como seguir las siguientes instrucciones:

 

1) Incluye SGW+DMX en el código fuente de tu juego después de incluir el propio Parser de InformATE!, pero antes del archivo de Gramática. Ejemplo:

 

Include "EParser";
Include "sgw+dmx";       ! Incluir la extension SGW+DMX
Include ">aventura.bli"; ! Incluir los recursos (imagenes y sonidos)
Include "Acciones";
!--------------------
! ... mas codigo ...
!--------------------
Include "Mensajes";
Include "Gramatica";

 

SGW+DMX utiliza la extensión “infglk.h” para compilar correctamente. Viene incluida en el paquete de SGW+DMX, así que recuerda copiarla en el directorio de tus librerías para InformATE!

 

 

2) En la rutina Inicializar(), llama a la rutina initializeSGW(x) para indicarle a SGW+DMX que prepare las ventanas gráficas y el sistema de audio. El argumento x corresponde a la altura que deberá tener la ventana superior para imágenes grandes (por ejemplo, para las imágenes de localidad). Ejemplo:

 

initializeSGW(240);

 

Esto va a configurar el tamaño de la ventana superior para permitir mostrar imágenes de “idealmente” 240px de altura. No es obligatorio que la imagen realmente tenga esa altura, pues si tiene una altura mayor que la indicada para la ventana entonces dicha imagen será “redimensionada” para ajustarse a la altura actual de la ventana.

 

Si indicas altura cero, es decir: initializeSGW(0), entonces SGW+DMX se inicializará correctamente, pero sin abrir la ventana gráfica superior. Podrás abrirla tú mismo más adelante, si así lo deseas, mediante la rutina openGraphicWindow().

 

NOTA: Si vas a ocupar Damusix, la rutina initializeSGW() también se encargará de inicializar el Gestor de Audio, ejecutando automáticamente la rutina Damusix.InicializarGlk(), por lo que no tienes que preocuparte de hacerlo tú mismo, a mano, en la rutina Inicializar() de tu juego. SGW+DMX lo hara por ti.

 

(En InformATE!, el Gestor de Damusix debe ser inicializado antes de poder trabajar con él. Usualmente esto lo debería hacer de forma manual el propio programador, llamando a Damusix.InicializarGlk() desde la rutina Inicializar() de su juego. Sin embargo, SGW+DMX intentar hacer las cosas todavía más simples, inicializando automáticamente el Gestor de Damusix, así que el programador no debe preocuparse de estos asuntos.)

 

 

3) Una vez que SGW+DMX esté inicializada ya puedes intentar reproducir un sonido o mostrar alguna imagen. La rutina playSound() sirve para tocar la música y sonidos. Para mostrar imágenes puedes usar las siguientes rutinas:

 

viewImageCenter(img) -> Imagen centrada en la ventana grande
viewImageRight(img)  -> Imagen a la derecha en la ventana grande
viewImageLeft(img)   -> Imagen a la izquierda en la ventana grande
viewImageSlide(img)  -> Imagen con deslizamiento en la ventana pequeña
closeImageSlide()    -> Si la ventana pequeña esta abierta, la cierra

 

En ninguna de estas rutinas es necesario especificar el tamaño de la imagen. Lo averiguaran por ellas mismas.

 

 

4) Puedes configurar los colores que quieres que tenga el juego. Simplemente tienes que declarar las siguientes constantes antes de incluir SGW+DMX:

 

Constant SCBACK = $110101;      ! estilo fondo
Constant SCTEXT = $DDBB99;      ! estilo texto
Constant SCEMPH = $FFFFDD;      ! estilo enfasis
Constant SCHEAD = $EEDDAA;      ! estilo cabecera (nombre localidades)
Constant SCINPU = $DDEEAA;      ! estilo input del jugador
Constant SCUSR1 = CLR_GG_BLUE;  ! estilo configurable User1
Constant SCUSR2 = CLR_GG_PINK;  ! estilo configurable User2

 

Opcionalmente, puedes indicar colores personalizados también para la Línea de Estado del juego:

 

Constant LIBACK = $30FF30;  ! estilo fondo linea de estado
Constant LITEXT = $0000A0;  ! estilo texto linea de estado

 

Los colores deben estar expresados en formato RRGGBB, mediante un número hexadecimal. Si no sabes el valor hexadecimal de algún color, puedes averiguarlo fácilmente con cualquier programa de retoque fotográfico. Hay una serie de constantes fáciles para los colores usados con mayor frecuencia:

 

CLR_GG_AZURE       CLR_GG_BLACK
CLR_GG_BLUE        CLR_GG_BROWN
CLR_GG_CYAN        CLR_GG_GREEN
CLR_GG_GREY        CLR_GG_MAGENTA
CLR_GG_ORANGE      CLR_GG_PINK
CLR_GG_PURPLE      CLR_GG_RED
CLR_GG_YELLOW      CLR_GG_WHITE

 

 

5) Al mostrar mensajes de texto puedes usar diferentes estilos. Ejemplos:

 

print "Estilo de Texto: ", (s_emph) "Emphasized", "^";
print "Estilo de Texto: ", (s_bold) "Bold", " (como Máquina-Z)^";
print "Estilo de Texto: ", (s_pref) "Preformatted", "^";
print "Estilo de Texto: ", (s_fixed) "Fixed", " (como Máquina-Z)^";
print "Estilo de Texto: ", (s_head) "Header", "^";
print "Estilo de Texto: ", (s_subhead) "Subheader", "^";
print "Estilo de Texto: ", (s_alert) "Alert", "^";
print "Estilo de Texto: ", (s_reverse) "Reverse", " (como Máquina-Z)^";
print "Estilo de Texto: ", (s_note) "Note", "^";
print "Estilo de Texto: ", (s_underline) "Underline/Italic", " (como Máquina-Z)^";
print "Estilo de Texto: ", (s_block) "BlockQuote", "^";
print "Estilo de Texto: ", (s_input) "Input", "^";
print "Estilo de Texto: ", (s_user1) "User1", "^";
print "Estilo de Texto: ", (s_user2) "User2", "^";

 

 

6) Si no quieres ocupar gráficos simplemente tienes que declarar la siguiente constante antes de incluir SGW+DMX:

 

Constant SGW_SIN_GRAFICOS;

 

Ahora tu juego sólo tendrá sonido, sin capacidades gráficas.

 

 

7) Si quieres usar Damusix para gestionar el audio, tienes que declarar la siguiente constante antes de incluir SGW+DMX:

 

Constant SGW_CON_DAMUSIX;

 

Ahora SGW+DMX incluirá e inicializará automáticamente la extensión Damusix. Todas las funciones nativas de audio de SGW+DMX serán inhabilitadas con este cambio y ya podrás usar sin problemas las rutinas de Damusix. Además, SGW+DMX hará todos los ajustes necesarios en el Marco de Trabajo (ver siguiente ítem) para que se usen las rutinas de Damusix que son mucho más potentes.

 

 

8) SGW+DMX implementa automáticamente un simple “Marco de Trabajo” que te hará la vida mas fácil a la hora de programar tu juego. El Marco de Trabajo mostrará una Imagen y tocará una Música de Fondo para cada Localidad de manera automática. También mostrará una imagen para cada Objeto que sea “examinado” durante el juego.

 

Para que una Localidad u Objeto muestren su Imagen, deben tener una propiedad sgw_img con la imagen que deseas.

 

Para que una Localidad toque su Música de Fondo, debe tener una propiedad sgw_mus con la música que deseas. Opcionalmente puedes indicar un volumen para aquella música, incluyendo una propiedad sgw_vol con el porcentaje de volumen (0-100) que deseas. Hay tres constantes fáciles que puedes usar también: VOLUMEN_ALTO (100%), VOLUMEN_MEDIO (75%), VOLUMEN_BAJO (50%). Ejemplos:

 

Object La_Localidad "Una Localidad Cualquiera"
  has luz,
  with
    sgw_img grafico_localidad, ! el grafico de la localidad
    sgw_mus musica_localidad,  ! la musica de la localidad
    sgw_vol VOLUMEN_ALTO,      ! el volumen de la musica
    descripcion "Yeah. Imagen y Musica de Fondo.",
;

Object El_Objeto "Un Objeto Cualquiera"
  with
    sgw_img grafico_objeto, ! el grafico del objeto
    descripcion "Cool. Este objeto tiene una Imagen.",
;

 

Si una localidad no tiene luz, el Marco de Trabajo no mostrará su imagen, limitándose sólo a dibujar la ventana gráfica vacía. Sin embargo, opcionalmente puedes indicar una imagen que sea representativa de “la oscuridad”, y el Marco de Trabajo la mostrará en cada localidad que no esté iluminada. Para usar este comportamiento, debes declarar la siguiente variable global antes de incluir SGW+DMX, asignándole el valor de la imagen que representará “la oscuridad”:

 

Global SGW_IMAGEN_OSCURIDAD = OSCURIDAD_TETRICA;

 

En este ejemplo, OSCURIDAD_TETRICA será la imagen que se va a mostrar cuando las localidades no tengan iluminación. Como SGW_IMAGEN_OSCURIDAD es una variable global, nada te impide asignarle una nueva imagen en cualquier otra parte del juego.

 

El Marco de Trabajo hace uso de las rutinas TrasElPrompt() y RutinaPostJuego(). Si necesitas implementar una versión propia de estas rutinas en tu juego, simplemente escríbelas antes de incluir SGW+DMX. No olvides que debes llamar en alguna parte de ellas a la rutina “gancho” del Marco de Trabajo. Así, dentro de TrasElPrompt() debes llamar a SGW_MarcoDeTrabajo(1) y dentro de RutinaPostJuego() debes llamar a SGW_MarcoDeTrabajo(2). Con esto, el Marco de Trabajo seguirá funcionando normalmente. Ejemplos:

 

[ TrasElPrompt ;
    SGW_MarcoDeTrabajo(1);
    !---------------------------------------------------------
    ! ... todo el codigo tuyo que necesites poner aqui ...
    !---------------------------------------------------------
];

[ RutinaPostJuego ;
    SGW_MarcoDeTrabajo(2);
    !---------------------------------------------------------
    ! ... todo el codigo tuyo que necesites poner aqui ...
    !---------------------------------------------------------
    rfalse; ! IMPORTANTE: SIN ESTO NO SE MUESTRA NINGUN TEXTO
];

 

Si por algún motivo necesitas escribir estas rutinas después de incluir SGW+DMX (por ejemplo, si estás copiando el mismo código que implementa SGW+DMX para el Marco de Trabajo, pero levemente modificado a manera de “hackeo”), entonces deberás hacer un Replace TrasElPrompt o un Replace RutinaPostJuego antes de incluir SGW+DMX, según sea la rutina que lo necesite.

 

Por otro lado, si no quieres usar el Marco de Trabajo sólo debes declarar la siguiente constante antes de incluir SGW+DMX:

 

Constant SGW_SIN_MARCO_DE_TRABAJO;

 

Ahora SGW+DMX compilará sin el Marco de Trabajo, lo cual sería algo muy triste, la verdad... =P

 

Información Técnica Adicional (no es necesaria para usos básicos):

 

Si compilas con Damusix, el Marco de Trabajo reproducirá la música de fondo de la localidad siempre en el último canal del Gestor de Audio; vale decir, en el canal 9 (DAMUSIX_NCANALMAX-1). Así, por ejemplo, si en un momento determinado quieres detener la música de fondo, tendrás que ejecutar Damusix.PararCanal(9). El mismo resultado lo puedes conseguir ejecutando Damusix.Parar(localizacion.sgw_mus), que es más intuitivo, pero entonces deberás tener la seguridad de que la localidad actual dispone de una propiedad sgw_mus y de que su música de fondo indicada ya se ha puesto en reproducción (esto es justo después de mostrar el Prompt luego de llegar a la localidad). Damusix.PararTodo() también puede serte útil si en el momento en que la llamas solamente está sonando la música de fondo. =P

 

Por otro lado, si utilizas el Marco de Trabajo hay un hecho que debes tener presente: como Damusix cuenta con un sistema de Volumen Global, el volumen de la música de fondo para cada localidad (es decir, la propiedad sgw_vol) siempre será “recalculado”, tomando como base el porcentaje de Volumen Global del Gestor de Damusix que se usará como valor máximo. Por ejemplo, si has puesto el valor VOLUMEN_ALTO (o 100%), se usará el valor actual del Volumen Global de Damusix; si has puesto VOLUMEN_MEDIO (o 75%), se usarán “tres cuartos” del valor actual del Volumen Global; si has puesto VOLUMEN_BAJO (o 50%), se usará “la mitad” del valor actual del Volumen Global; y así por el estilo. En resumen, el valor del volumen para las propiedades sgw_vol siempre será recalculado a partir de un máximo establecido por el valor actual del Volumen Global de Damusix.

 

 

9) NOTA TÉCNICA: La extensión SGW+DMX ocupa los siguientes Puntos de Entrada Glk para funcionar correctamente: IdentifyGlkObject() y HandleGlkEvent(). Si necesitas implementar una versión propia de estas rutinas en tu juego, simplemente escríbelas antes de incluir SGW+DMX. Pero no debes olvidar algo MUY IMPORTANTE:

 

- Dentro de IdentifyGlkObject() debes llamar a:
    SGW_IdentifyGlk(fase,tipo,ref,rock);

- Dentro de HandleGlkEvent() debes llamar a:
    SGW_HandleGlk(ev);

 

Ejemplos:

 

[ IdentifyGlkObject fase tipo ref rock;
    ! IdentifyGlkObject de libreria SGW
    SGW_IdentifyGlk(fase,tipo,ref,rock);
    !------------------------------------------------------------------
    ! ESTA LINEA ES NECESARIA SOLO SI ESTAS USANDO DAMUSIX:
    ! Damusix.IdentificarSonidos(fase);
    !------------------------------------------------------------------
    !------------------------------------------------------------------
    ! ... todo el codigo tuyo que necesites poner aqui ...
    !------------------------------------------------------------------
];

[ HandleGlkEvent ev;
    ! HandleGlkEvent de libreria SGW
    SGW_HandleGlk(ev);
    !------------------------------------------------------------------
    ! ESTA LINEA ES NECESARIA SOLO SI ESTAS USANDO DAMUSIX:
    ! Damusix.NotificarFade(ev); ! solo si usas Fades en "tiempo-real"
    !------------------------------------------------------------------
    !------------------------------------------------------------------
    ! ... todo el codigo tuyo que necesites poner aqui ...
    !------------------------------------------------------------------
];

 

Con esto, SGW+DMX funcionará normalmente y podrás programar tus propias versiones de los Puntos de Entrada Glk mencionados.

 

 

Funciones Gráficas

 

testGraphics(flag_aviso) (Rutina disponible sólo para Máquina Glulx)

Averigua si el intérprete Glulx en el que se está ejecutando el juego puede dibujar gráficos. Si el argumento flag_aviso vale 1, entonces se mostrará un mensaje de aviso apropiado indicando que el intérprete no soporta gráficos, si fuera el caso. Esta rutina devolverá true si hay soporte para gráficos y false si no lo hay.

 

initializeSGW(x)

Inicia todo lo necesario dentro en la Máquina Glulx para que SGW+DMX pueda funcionar. Abre las ventanas gráficas y prepara el sistema de audio. El argumento x indica el alto en píxeles que debe tener la ventana gráfica superior (idealmente, no mayor a 240px). Si el argumento x vale cero, entonces SGW+DMX iniciará correctamente, pero sin abrir la ventana gráfica superior; posteriormente, el programador puede abrirla, si lo desea, mediante la rutina openGraphicWindow(). Importante: la presente rutina debe ser llamada solamente desde el interior de la rutina Inicializar(). Información adicional: mirar Nota sobre el Tamaño de las Ventanas Gráficas. (NOTA CON DAMUSIX: esta rutina inicializará automáticamente el Gestor de Damusix, por lo que no es necesario que el programador haga una llamada a Damusix.InicializarGlk() desde la rutina Inicializar() del juego.)

 

clearMainWindow()

Limpia el contenido de la ventana principal del juego (es decir, limpia la ventana del buffer de texto y la ventana gráfica superior).

 

clearGraphicWindow()

Limpia el contenido de la ventana gráfica superior.

 

clearTextWindow()

Limpia el contenido de la ventana del buffer de texto.

 

clearStatusLine()

Limpia el contenido de la ventana de la línea de estado.

 

closeAllWindows()

Cierra todas las ventanas gráficas abiertas.

 

openGraphicWindow(x)

Abre la ventana gráfica superior. El argumento x indica el alto en píxeles que debe tener la ventana (idealmente, no mayor a 240px). Si la ventana ya está abierta, la cierra y la vuelve abrir con el alto especificado. Información adicional: mirar Nota sobre el Tamaño de las Ventanas Gráficas.

 

closeGraphicWindow()

Cierra la ventana gráfica superior si estuviera abierta. En caso contrario, no se hace nada.

 

viewImageCenter(img)

Muestra una imagen en la ventana superior con posición centrada.

 

viewImageLeft(img)

Muestra una imagen en la ventana superior con posición a la derecha.

 

viewImageRight(img)

Muestra una imagen en la ventana superior con posición a la izquierda.

 

viewImageSlide(img,flag_deslizar) (Esta rutina hace uso del Timer Glk)

Muestra una imagen en la ventana gráfica pequeña (para objetos) con un bonito efecto de deslizamiento de entrada, luego espera unos segundos y finalmente cierra la ventana con un deslizamiento de salida. Si el argumento flag_deslizar vale 1, entonces el efecto de deslizamiento no se producirá; en tal situación, el programador deberá cerrar por sí mismo la ventana, mediante la rutina closeImageSlide(), o esperar que la cierre automáticamente el Marco de Trabajo (si se estuviera usando) en el siguiente turno de juego. El tamaño de las imágenes mostradas en esta ventana debería ser idealmente no mayor a 200px de alto. Información adicional: mirar Nota sobre el Tamaño de las Ventanas Gráficas. (NOTA CON DAMUSIX: si un efecto de Fade en “tiempo-real” está activo cuando se ocupa esta rutina, entonces siempre se forzará el no-deslizamiento para no interrumpir el Fade, que también usa el Timer Glk.)

 

closeImageSlide()

Cierra la ventana gráfica pequeña (para objetos) si estuviera abierta. En caso contrario, no se hace nada.

 

Nota sobre el Tamaño de las Ventanas Gráficas: las rutinas que dibujan gráficos en la ventana grande y pequeña dejarán por defecto un borde de 10px alrededor de la imagen (sólo por estética), por lo que la imagen se dibujaría siempre con 20px menos de altura. Para compensar esta variación y que la imagen se dibuje realmente con el tamaño que le corresponde, siempre se sumarán 20px al propio tamaño de la ventana gráfica. Por ejemplo, si abres una ventana superior de 100px, entonces se creará dicha ventana con una altura efectiva de 120px realmente. Con estos 20px adicionales, la imagen ya tendrá espacio para dibujarse con la altura “verdadera” que indicamos para la ventana. De la misma manera, la ventana pequeña siempre toma el ancho de la imagen que va a mostrar y le suma 20px más (por ejemplo, si tenemos una imagen de 50px de ancho, la ventana se abrirá con 70px de ancho para compensar el borde). Si no te gusta este comportamiento puedes definir la constante BORDEWIN con valor cero antes de incluir SGW+DMX; o si quieres un borde más o menos grueso, puedes poner un valor a tu gusto. Ejemplos:

 

Constant BORDEWIN = 0;  ! para no tener borde alrededor de las imagenes, o...
Constant BORDEWIN = 40; ! para tener un borde de 40px alrededor de las imagenes

 

 

Funciones de Audio

 

testSounds(flag_aviso) (Rutina disponible sólo para Máquina Glulx)

Averigua si el intérprete Glulx en el que se está ejecutando el juego puede reproducir sonidos. Si el argumento flag_aviso vale 1, entonces se mostrará un mensaje de aviso apropiado indicando que el intérprete no soporta sonidos, si fuera el caso. Esta rutina devolverá true si hay soporte para sonidos y false si no lo hay.

 

Las siguientes funciones nativas de audio de SGW+DMX estarán inhabilitadas si se compila usando la extensión Damusix. Tenga presente el programador que no se podrá ejecutar ninguna de estas rutinas, pues simplemente no existirán.

 

playSound(canal,sonido,repeticiones,volumen)

Toca un sonido o música en el canal de audio indicado. El canal debe ser alguno de los tres canales disponibles: music (para la música), chan1 (para efectos de sonido 1) o chan2 (para efectos de sonido 2). El argumento repeticiones debe ser siempre un valor mayor a cero, con la excepción del valor especial -1 que indica “repeticiones infinitas”. El volumen se expresa siempre en porcentaje (0 a 100).

 

setVolume(volumen,canal)

Cambia el volumen del canal de audio indicado. El volumen se expresa siempre en porcentaje (0 a 100).

 

silenceChannel(canal)

Detiene el sonido del canal de audio indicado.

 

silenceAll()

Detiene todos los sonidos (es decir, detiene los canales music, chan1 y chan2).

 

 

Otras Funciones

 

wait("texto",tiempo_espera) (En modo “temporizado”, esta rutina hace uso del Timer Glk)

Espera hasta que se pulse una tecla. Los dos argumentos de la rutina son completamente opcionales. Si se le pasa un texto, lo mostrará mientras espera la pulsación. Si se indica un tiempo de espera límite, entonces esperará hasta que se pulse una tecla o se acabe el tiempo, lo que suceda primero. El tiempo de espera debe expresarse en décimas de segundo. Si no se quiere mostrar texto, pero sí usar un tiempo de espera, la llamada deber tener la siguiente forma: wait(0,tiempo_espera). Esta rutina devolverá el valor del carácter de la tecla pulsada (1 a 255) o cero si durante una espera temporizada se agotó el tiempo y no se pulsó ninguna tecla. (NOTA CON DAMUSIX: si un efecto de Fade en “tiempo-real” está activo cuando se ocupa esta rutina, entonces siempre se forzará la no-temporización para no interrumpir el Fade, que también usa el Timer Glk.)
